## 1B. IS-LM vs IS-MP · 双轨教学策略 🆕(回应教材版本变化)

**关键事实**:米什金第 11 版起已**改用 IS-MP 曲线代替 LM 曲线**推导 AD。原因:1980s 后各大央行从盯货币量(M2 增速)转向盯利率(联邦基金利率),LM 推导前提"M^s 外生固定"已与现实脱节。**MP 曲线** = "央行根据 π 设定 i" 的反应函数(类似泰勒规则)。

但 LM 在教学上仍有不可替代的价值——它能让学生**直观看到"利率 / 货币量 / 产出"三者相互决定**,而 MP 曲线把"央行设利率"作为 black box,跳过了机制。所以本章策略是双轨混合,各取所长。

### 1B.1 双轨混合表

| 阶段 | 用什么 | 原因 |
|---|---|---|
| §1b 概念引入 | LM | 简单、直观、好画 |
| §2 模拟器主图(默认) | LM(双线交点) | 沙盘可视化最清晰 |
| §3 主线题(q001-q010) | LM | 难度低 |
| §3 进阶题 q011-q013 | LM | 同上 |
| §3 进阶题 [ch20-q014/q015](08_种子题库_v0.md) | **MP**(明确对照 LM) | 让学生知道现实里央行是怎么操作的 |
| §4 新闻三联单(关联 Fed 决议) | MP(因为 Fed 决议是"利率决议") | 贴现实 |
| §6 AI 答疑(用户问"Fed 怎么决策") | MP 切换 | 用户主动问到时露出 |

### 1B.2 关键时机:何时把 MP 介绍给学生

**不在第一次接触就讲**——MP 比 LM 复杂(它是央行反应函数 + 通胀方程的组合,新手会被吓到)。

**最佳时机**:用户已通过模拟器把 LM 玩熟,然后第 [ch20-q014](08_种子题库_v0.md) 题发起"教材实际用什么?"的反问,触发 1 分钟微课对照两个版本。这一安排同时满足:

- 学习曲线平滑(LM 先,MP 后)
- 教材忠实(让学生知道现实版本)
- 反预期触发(已熟悉 LM 的用户在 q014 被反问)

### 1B.3 模拟器需要支持的"两种模式"(影响开发)

§2.2 沙盘 UI 加一个 toggle:

| 模式 | 主图 | 滑块 |
|---|---|---|
| **LM 模式**(默认) | IS + LM | a (IS 截距) / b (IS 斜率) / c (LM 截距) / d (LM 斜率) |
| **MP 模式**(进阶) | IS + MP(水平线代表央行设的 i*) | a / b / i_target / π_target / Fed 反应敏感度 |

**工作量影响**:LM 模式即 [04 §2 现有规格];MP 模式需要新增 toggle + 公式分支 + 滑块替换,**+1 人日**。MVP 可只做 LM 模式,MP 模式作为 v1.5 增量。

### 1B.4 ZLB 在两个模式下的呈现差异(教学高密度时刻)

- **LM 模式**:LM 在 i ≤ 0 处变水平段(用户拖 c 滑块到极端时自动出现)
- **MP 模式**:MP 在 i_target = 0 处水平,Fed 想再降也没空间

**关键**:两个模式都能演示流动性陷阱,但 MP 模式更贴近现实——它直白展示"Fed 想降但没法降"的政策无效。这是 [ch20-q015](08_种子题库_v0.md) 直接利用的反预期点。

## 2. IS-LM 沙盘(P0 模拟器)🔧(框架复用,主图 = 二维曲线)

### 2.1 数学模型

**IS 曲线**(产品市场均衡):

```
Y = C(Y - T) + I(i) + G + NX(e, Y)
```

简化形式(线性):
```
Y = (1/(1-c(1-t))) × [C₀ + I₀ - b·i + G + NX₀]
→ IS:Y 与 i 反向(斜率 -1/b)
```

**LM 曲线**(货币市场均衡):

```
M^s / P = L_y · Y - L_i · i
→ LM:Y 与 i 同向(斜率 L_y/L_i)
```

**联立求均衡**:

```
i* = [(C₀ + I₀ + G - (1-c(1-t))·M^s/P / L_y × ... ) / ...]
Y* = ...
```

UI 不需要展示完整代数推导,**让用户用"拖动 IS / LM 看新均衡"代替解方程**——这是 IS-LM 模型作为教学工具的核心价值。

## 12. 风险与开放问题 🆕

1. **模型简化的诚实标注**:IS-LM 是 1937 年 Hicks 提出的简化模型,主流宏观早已用 DSGE / NK 模型。MVP 是否要加一个"模型局限性"卡片告诉用户"这是教学工具而非现实模型"?(我倾向加,放在 §6 答疑)
2. **流动性陷阱实证争议**:学术界对 2010-2015 是否真的是陷阱有分歧。MVP 建议:不下定论,但展示"陷阱视角下的解释"+"非陷阱视角"的对比卡
3. **跨章题的难度爬升**:第 7、10 题要求同时调用三章知识,初学者卡住率会很高。建议:这两道题加"前置知识检查"(没学完 14、17 章时不出现这两道,而是出"单章版本")
4. **沙盘的视觉复杂度**:四象限 + 多曲线在小程序屏幕上很挤。MVP 建议:默认只显示主图,副图通过侧边 tab 切换,避免一屏太满

### 15.3 误区底库(三章合计 24 条)的合并机会

读完三章误区库,可以发现至少 3 组跨章共享模式:

1. **常数误区**:MM-03(m 是常数)+ ISLM-02(LM 总上斜)+ IRP-01(高息必升值)
   → 合并成"金融模型的参数变化是机制本身"专项微课
2. **政策万能误区**:MM-08(QT 销毁钱)+ ISLM-04(货币万能)+ IRP-04(央行干预万能)
   → 合并成"政策的传导边界"微课
3. **教科书 vs 现实差异**:MM-01(印钱必通胀)+ ISLM-03(财政必挤出)+ IRP-05(CIP 永远成立)
   → 合并成"模型成立条件"微课

**这 3 个跨章微课可以做成"高级用户"的彩蛋内容**——只对完成三章的用户开放,作为"宏观经济直觉跃迁"的一次性奖励。

### 15.4 模板验证结论

跑完三章可以确认:

1. **14 节模板对宏观存款机制(微观银行)/ 国际金融(开放经济)/ 总量均衡(纯理论)三种差异极大主题都成立** —— 模板不是为某一章定制
2. **机制层复用度稳定在 90%+,内容层必须全新** —— 这是宿命,不是模板问题
3. **真正的下一步瓶颈是"内容流水线"**:教研 + 题库 + 误区 + 数据 + 案例,五个独立工种,不是开发瓶颈
4. **跨章联动是产品的护城河** —— 单章学习产品很多,但能让用户"做完一道题感受到三章打通"的极少。这是 §3.3 第 10 题、§15.3 跨章微课要重点投入的方向
