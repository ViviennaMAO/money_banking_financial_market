# MVP 章节拆解:第 14 章 货币乘数(端到端样板)

> **目的**:这一份文档把第 14 章在 Luffa SuperBox 小程序里要做的所有东西讲清楚——交互模拟器、题库、新闻三联单、看板联动、AI 答疑、社交分发、数据需求。第 17 章(利率平价)和第 20 章(IS-LM)直接复用这个模板。

## 0. 学习科学锚点(v1 设计原则)

本章所有交互都基于以下 6 条经过验证的学习科学原理。每个模块都标注了它服务于哪一条——这些不是装饰,而是反向设计的依据:

| # | 原理 | 一句话 | 体现在本章 |
|---|---|---|---|
| 1 | **测试效应** | 先测后讲 > 先讲后测;凭直觉答错的题在揭晓答案后记得最牢 | §1a 把"前测 3 题"放在所有教学之前 |
| 2 | **生成效应** | 自己生成答案 > 被动看到答案 | §2.7 预测先行;§2.9 自解释输入 |
| 3 | **反预期** | 直觉被打破的瞬间记忆最深 | §2.4 "乘数从 8.7 砸到 1.2";§3.3 第 10 题 |
| 4 | **必要难度** | 略高于舒适区的挑战留下记忆;太顺反而过目即忘 | §2.8 挑战模式;§3.4 误区诊断 |
| 5 | **自解释** | 能用自己的话讲一遍 = 真的懂 | §2.9;§3.7 "教别人"收尾 |
| 6 | **元认知** | 知道"自己知道什么和不知道什么" | §14 个性化路径;每节后的自评 |

**设计 KPI**:用户在 §1a 前测和 §3.5 后测之间的**提升幅度**(而不是"通过率")。我们要的是真的学会,不是跑完流程。

## 2. 货币乘数实验室(P0 模拟器规格)⚙️

### 2.1 数学模型

**核心公式**(米什金第 14 章):

```
m = (1 + c) / (r + e + c)

M2 = m × MB
```

其中:
- **MB**(基础货币 / Monetary Base)= 流通中现金 + 银行准备金
- **r**(法定准备金率)= 法定准备金 / 存款
- **e**(超额准备金率)= 超额准备金 / 存款
- **c**(现金存款比)= 流通中现金 / 存款

**重要现实修正**(必须在 UI 里给用户看到):
> 美国自 2020 年 3 月起 r = 0,公式实际退化为 `m = (1+c)/(e+c)`。中国仍保留 RRR(法定准备金率)。这个差异本身就是教学点。

### 2.5 派生存款动画

按"▶"按钮 → 一笔 $1000 初始存款,5 轮派生:

```

  1    $1000        $120             $880

```

### 2.6 边界与坑

- **分母为 0 保护**:r+e+c=0 时 UI 显示"乘数无穷大,意味着银行系统没有任何漏出——现实中不可能"
- **理论 vs 实测**:右下角永远显示"实测 m = M2/MB = 当前真实值",和滑块算出的"理论 m"对照——两者偏离时弹提示"理论假设所有银行行为齐次,实际有结构差异"
- **超储率上限说明**:e 调到 >30% 时弹"现实中只有 2008-2014 短暂出现过 e>50%,因为 IORB 给得太香"

### 2.7 预测先行(切换历史快照前的弹窗)🎯 生成效应 + 反预期

每次切换历史快照前,先弹一个 5 秒小窗:

```
你即将切换到「2010.1 QE 后」。
在看到答案之前,先猜一下:
  乘数 m 会变成多少?
  □ 跟现在差不多(5 左右)
  □ 涨到 10+(QE 注入了大量货币)
  □ 暴跌到 1-2(超储锁住了流动性)
[确认我的猜测]
```

确认后才显示真实数值。猜对/猜错都给反馈:
- 猜错(直觉派):"很多人和你想的一样——这就是为什么这一章值得学。"
- 猜对(已懂派):"你已经把这个机制内化了,试试 §2.8 挑战模式。"

**这个"被打脸"的瞬间是单一最高密度的学习时刻**。每个快照切换都必须加。

```

[___________________________]

≥ 7 分解锁徽章「能讲明白货币乘数的人」。低分给具体优化建议(不直接给答案)。

### 3.2 题目样例(第 5 题完整规格)

```
[第 5 题]  公式套用 · ⭐⭐⭐

Fed 启动一轮 QE,基础货币 MB 增加 $1 万亿。假设当前
r = 0%, e = 12%, c = 8%。

请问 M2 的理论增加量最接近?

  A. $1 万亿       B. $3 万亿
  C. $5 万亿       D. $10 万亿

[查看公式]  [打开模拟器试一试]  [提交]

────────────────────
答案:C(m ≈ 5.4,所以 ΔM2 ≈ $5.4 万亿)

解析:
  m = (1+0.08) / (0+0.12+0.08) = 1.08/0.20 = 5.4
  ΔM2 = m × ΔMB = 5.4 × $1T ≈ $5.4T

⚠️ 但现实中 2008-2014 实测 ΔM2/ΔMB 远小于理论值——
   因为 e 在 QE 期间从 0.04% 飙到 80%。
   下一题我们就来看这个现象。

[查看现实数据] → 跳看板  [下一题 →]
```

### 3.3 第 10 题(反常识题,亮点)

```
[第 10 题]  反常识 · ⭐⭐⭐⭐⭐

2020 年 3 月-2022 年 6 月,Fed 资产负债表从 $4.2T 扩张到 $9T,

  - 第 24 章:预期锚定(通胀预期解构)

### 3.4 误区诊断(取代简单的"错 2 次重看动画")🩺 必要难度 + 元认知

每道题的错项都绑定一个具体误区代码(见 §13 误区库)。错题不只是"错"——系统识别误区类型并定向修复:

```
你在第 5 题选了 D($10 万亿)
→ 误区代码 [MM-03] "把 m 当成固定常数"
→ 推送针对性 60 秒微课:m 的现实波动 + 2007 vs 2010 对比
→ 让你重做第 5 题的"变体"(改了数字,机制相同)
→ 仍错 → 标记为"持续混淆点",计入用户档案
```

**累计 3 个相同误区代码**触发"专项诊所":针对该误区的 5 分钟深度交互页。

### 3.5 后测(伪装在测验里的提升测量)📊 测试效应回路

§1a 前测的 3 道直觉题,在用户做完第 1-3 题时**伪装成新题**重新出现一次。系统对比:
- 前测对了多少 vs 后测对了多少
- 前测错的 3 道里,后测改对了几道

这个**"提升曲线"**才是产品的真 KPI(见 §0 学习科学锚点)。
反馈话术:"你在'M2 ≠ 印钞量'这一点上从错变对——这是本章最大收获。"

### 3.6 间隔复习(交错而非块状)🔁

错题不只 72 小时回来,还会:
- 与第 17 章利率平价、第 20 章 IS-LM 的题**交错**出现(interleaving)
- 第 14 章学完 7 / 14 / 30 天各推一道"老题翻新"(改数字、换情境)
- 推送时间随机化,模拟"突击考试"的检索压力

- 是否避免术语堆砌(给术语用日常类比)
- 是否触及核心机制(乘数 / V / 预期 至少 1 个)
- 是否在 60 秒内讲完

得分 ≥ 7 → 解锁稀有徽章「老师范儿」+ 录音/文字一键分享到 Channel,做"同学互讲"内容池。**这是社交分发最深的一勺**——别人讲明白的东西,你也讲一个,就是教学循环。

  [📖 翻到第 14 章]   [💱 在模拟器试一下]

  要让 M2 同比 +5%,Fed 需要重启每月多大规模 QE?

## 12. 风险与开放问题

1. **乘数公式的现实失效**:理论公式 `m=(1+c)/(r+e+c)` 在 IORB 时代已不太能精准预测 M2,要不要在 UI 里多大力度展示这个"理论失灵"?(我倾向:大力展示,这本身就是教学点;但视觉上不能让初学者觉得"那这公式还学它干嘛")
2. **派生存款动画 vs 现实**:动画讲的是教科书"全准备金"的简化版,现实中银行先放贷再找准备金。要不要在第 8 题考一下这个"颠倒"?
3. **中美对照的政治敏感度**:某些章节会涉及中国央行决策,内容尺度需要把握。MVP 建议 14 章先做"机制对比",不做"政策评论"。
4. **看板字段名我做了猜测**[待对齐]:e、c、m 这些字段你的看板里到底叫什么名字?第一次联调时务必对齐。

## 14. 个性化学习路径(基于诊断分流)🧭

§1a 前测 + §3.4 误区诊断 把用户分到三条路径之一,**学习时长一致(都 12 分钟),但难度密度和交互方式完全不同**:

**路径 A · 直觉错家**(前测 0-1 对)
→ 完整流程,§2.4 历史快照重点演示反预期震撼
→ 顺序:§1b(慢)→ §2 全部 → §3 每题带提示 → §4
→ 完成后:第 13 章(央行制度)← 先制度后理论

**路径 B · 半懂半混**(前测 2 对,但触发 2+ 个 MM 误区)
→ 跳过 §1b,§2 开启"教练模式"(每滑块都弹解释)
→ 强化 §3.4 误区诊所
→ 完成后:第 17 章(汇率)横向扩展

**路径 C · 已懂派**(前测全对)
→ 跳过 §1b 和 §2 自由探索,直接 §2.8 挑战模式
→ 测验只做反常识题(第 8、9、10)
→ 完成后:第 24 章(通胀理论)纵深 + §3.7 教别人

**关键设计**:同一份内容,根据元认知数据动态裁剪。SuperBox 单应用做不到"大而全"——这反而成了优势:**每个用户看到的是为他剪裁过的子集**,而不是面对全本教材的劝退感。
